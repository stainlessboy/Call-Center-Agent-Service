Agent Architecture · полный гайд

Что происходит внутри app/agent/ от момента, когда клиент отправил сообщение, до момента, когда бот ответил. Включая LangGraph, детекцию языка, PII-маскирование, инструменты LLM и checkpointing.

Обзор

Агент построен на LangGraph — state-machine из 4 узлов (router, faq, calc_flow, human_mode). LLM — ChatOpenAI из langchain-openai. Состояние сохраняется между turn'ами через checkpointer (Postgres в проде, Memory в dev). Язык пользователя детектится отдельным маленьким LLM-вызовом (или regex fast-path) до запуска графа. Вся персистентность — через SQLAlchemy async.

Главный принцип: LLM не определяет интент прямо — он видит 12 инструментов (find_office, get_products, faq_lookup, …) и сам выбирает, какой вызвать. Router только разводит режим (калькулятор / оператор / FAQ).

Схема графа

Маршрутизация — через Command(goto=...), а не conditional_edges. Каждый узел возвращает Command(update={...}) и LangGraph атомарно обновляет state.

Жизненный цикл запроса

16 стадий от Telegram до ответа. Номера на левой шкале соответствуют вертикальному порядку выполнения.

Telegram webhook → aiogram entry

Webhook попадает в handle_text() (commands.py:536). Substring-шорткаты убраны — любой произвольный текст идёт прямо в агент. Только нажатия кнопок меню (action == CURRENCY_RATES и т.д.) обрабатываются на уровне handler'а.

ChatService.handle_user_message db

  • chat_service.py:201-204 — получить/создать ChatSession, обновить last_activity.
  • chat_service.py:205-210 — записать user-сообщение в БД.
  • chat_service.py:241-254 — вызвать agent_client.send_message() с таймаутом, замер latency.

AgentClient → Agent.send_message wrapper

agent_client.py — тонкая обёртка, форвардит в agent.py:94.

Agent._ainvoke — загрузка state graph

  • agent.py:53 — config = {"configurable": {"thread_id": session_id}}.
  • agent.py:54 — _aload_existing_state() достаёт prior state из checkpointer.
  • agent.py:55 — извлечь dialog или _default_dialog().

Детекция языка · lang_detect.detect_language llm

  • lang_detect.py:209 — _should_skip_detection: если <2 символа или нет букв → вернуть fallback.
  • lang_detect.py:212 — fast-path: regex на uz-Cyrillic (ў/қ/ғ/ҳ) + _UZ_MORPHOLOGY_RE (олмокчи, рахмат, менга, \w+сизме…) → сразу "uz".
  • lang_detect.py:216-224 — иначе LLM-детектор (gpt-4o-mini, LANG_DETECTOR_MODEL) с PII-маскировкой на входе.

Результат пишется в state["lang"] и dialog["last_lang"].

Свежий SystemMessage под язык llm

  • agent.py:65 — get_system_policy(detected_lang) берёт per-language промпт.
  • agent.py:66-70 — первый SystemMessage в messages заменяется на свежий.

Без этого язык инструкций «замораживался» бы в checkpointer — при смене языка пользователем промпт оставался бы старым.

Сборка BotState graph

agent.py:72-85 — строится BotState со всеми полями: messages, last_user_text, answer, human_mode, keyboard_options, dialog, lang, session_id, user_id, show_operator_button, token_usage.

PII-маскирование llm

Применяется в двух точках:

  • lang_detect.py:223 — перед детектором.
  • nodes/helpers.py:37 — в _finalize_turn перед записью в историю.

Паттерны (pii_masker.py): EMAIL → IBAN (UZ…) → PHONE (+998) → CARD (16) → PINFL (14) → PASSPORT (2+7). Идемпотентно. Даты и ИНН не трогаются (чтобы не ломать суммы/сроки).

Graph.ainvoke → node_router graph

agent.py:86 — self._graph.ainvoke(state_in, config=config). Граф собран в graph.py.

node_router · маршрутизация graph

  • router.py:19-20 — human_mode=True → Command(goto="human_mode").
  • router.py:22-25 — lead_step set или flow==FLOW_CALC → calc_flow.
  • router.py:27-46 — кнопка «Рассчитать» + FLOW_PRODUCT_DETAIL + выбранный продукт → calc_flow.
  • router.py:48 — иначе → faq.

node_faq · LLM + 12 tools (≤3 раунда) llm

  • faq.py:220-221 — _normalize_user_text: collapse whitespace, trim !?..;:, limit 2000.
  • faq.py:228-244 — per-language system policy + XML со state + trim до MAX_DIALOG_MESSAGES=12.
  • faq.py:252-273 — цикл до 3 раундов:
    1. llm_with_tools.ainvoke(loop_msgs) — bind_tools(_FAQ_TOOLS).
    2. accumulate_usage() — копим токены.
    3. Если tool_calls пусто → финальный текст, выход.
    4. Иначе ToolNode(_FAQ_TOOLS).ainvoke(...) выполняет все вызовы, результаты в loop_msgs.
  • faq.py:274-280 — catch на APIError/TimeoutError/JSONDecodeError: fallback _faq_lookup().
  • faq.py:287 — _update_dialog_from_tools() обновляет dialog по последнему tool_call.

node_calc_flow · расчёт + захват контакта graph

calc_step: extract_calc_value() (calc_extractor.py) — LLM-парсинг числа или side-question; clamp к ограничениям продукта; следующий невыполненный вопрос; финал — PDF-график (кредит) или текст (вклад).

lead_step: offer → name → phone. На phone вызывается _save_lead_async() → запись Lead в БД, dialog сбрасывается.

node_human_mode · interrupt graph

human_mode.py:11 — langgraph_interrupt(...) приостанавливает граф. Agent.resume_human_mode() передаёт Command(resume=operator_reply).

_finalize_turn · оформление ответа output

  • Маскировка user-текста (или явный токен [PHONE]/[NAME] для lead_step).
  • Добавление HumanMessage + AIMessage в messages, trim до MAX_DIALOG_MESSAGES.
  • fallback_streak: если ≥3 или _is_operator_request() → show_operator_button=True.
  • Возврат dict с messages/answer/dialog/keyboard/show_operator_button/token_usage.

Пост-граф → AgentTurnResult output

agent.py:87-92 — выход графа оборачивается в AgentTurnResult(text, keyboard_options, show_operator_button, token_usage).

ChatService persist + Telegram reply output

chat_service.py:274-295 — парсинг маркера [[PDF:...]], запись assistant-сообщения с latency и token_usage, возврат AgentReply.

Aiogram-handler отправляет text + flow_keyboard + operator_button пользователю.

Дерево модуля app/agent/

app/agent/
├── agent.py              # Agent class — entry point, send_message()
├── graph.py              # StateGraph(router → faq|calc_flow|human_mode → END)
├── state.py              # BotState TypedDict + AgentTurnResult
├── llm.py                # ChatOpenAI factory + token usage/cost
├── tools.py              # 12 @lc_tool functions + _FAQ_TOOLS list
├── constants.py          # CREDIT_SECTION_MAP, flow/step names, VALID_LANGS
├── i18n.py               # Per-language system policy + UI strings
├── intent.py             # Rule-based intent helpers (_is_greeting, _is_calc_trigger…)
├── products.py           # Load/format/match products from DB
├── calc_extractor.py     # LLM-based numeric extraction for calculator
├── parsers.py            # Response parsing utilities
├── checkpointer.py       # memory | postgres async checkpointer
├── lang_detect.py        # fast-path regex + LLM detector
├── pii_masker.py         # PII regex masker (EMAIL/IBAN/PHONE/CARD/PINFL/PASSPORT)
└── nodes/
    ├── router.py         # node_router — routes to faq/calc_flow/human_mode
    ├── faq.py            # node_faq — LLM with tool-calling (≤3 rounds)
    ├── calc_flow.py      # node_calc_flow — calculator + lead capture
    ├── human_mode.py     # node_human_mode — interrupt() for operator handoff
    └── helpers.py        # _finalize_turn, _save_lead_async

Справочник файлов

agent.py
главный класс
entry point
Класс Agent, инкапсулирующий граф и жизненный цикл state.
  • Agent.setup(backend, url) — async-checkpointer (Postgres/Memory).
  • Agent._ainvoke(...) — детекция языка, загрузка prior state, свежий SystemMessage, сборка BotState, graph.ainvoke.
  • Agent.send_message(...) — публичный API.
  • Agent.resume_human_mode(...) — Command(resume=...) после interrupt.
  • Agent.sync_history(...) — синхронизация истории из внешнего источника.
graph.py
LangGraph · compile
Построение и компиляция StateGraph.
  • build_graph(checkpointer, store) — 4 узла: router → [faq|calc_flow|human_mode] → END.
  • Маршрутизация — Command(goto=...), а не add_conditional_edges.
state.py
types · схема
TypedDict и dataclass для структуры состояния диалога.
  • BotState — messages, last_user_text, answer, human_mode, keyboard_options, dialog, lang, _route, session_id, user_id, show_operator_button, token_usage.
  • AgentTurnResult — финальная структура ответа.
  • _default_dialog() — initializer для dialog dict.
llm.py
LLM factory + токены
ChatOpenAI instance, подсчёт токенов, расчёт стоимости.
  • _get_chat_openai() — кэшированный ChatOpenAI (поддержка reasoning_effort для gpt-5, responses API).
  • extract_text_content, extract_token_usage — нормализация разных форматов ответа.
  • accumulate_usage, finalize_usage, calculate_cost — учёт по _MODEL_PRICING.
tools.py
12 tools · LLM API
12 @lc_tool-инструментов в _FAQ_TOOLS. Все, кому нужен dialog, объявляют state: Annotated[dict, InjectedState] = None — ToolNode инжектит state, параметр скрыт от LLM.
  • greeting_response() — привет + меню.
  • thanks_response() — «пожалуйста».
  • find_office(office_type, query) — филиалы/офисы.
  • get_office_types_info() — объяснение типов офисов.
  • get_currency_info() — курсы ЦБ.
  • show_credit_menu() — меню типов кредита.
  • get_products(category) — список продуктов.
  • select_product(product_name) — детали продукта (3-tier fuzzy).
  • start_calculator() — переход в calc_flow.
  • custom_loan_calculator(amount, term, downpayment) — расчёт с фикс. ставкой.
  • faq_lookup(query) — поиск в FAQ-базе.
  • request_operator(reason) — передача оператору.
constants.py
константы · маппинги
Конфигурационные константы: языки, категории, пороги.
  • VALID_LANGS = ("ru","en","uz"), FALLBACK_STREAK_THRESHOLD=3.
  • Flow: FLOW_SHOW_PRODUCTS, FLOW_PRODUCT_DETAIL, FLOW_CALC.
  • Шаги калькулятора: STEP_AMOUNT/TERM/DOWNPAYMENT.
  • CREDIT_SECTION_MAP — category → section_name в БД.
i18n.py
переводы · промпты
Локализация (ru/en/uz) для системного промпта и UI строк.
  • get_system_policy(lang) — длинный per-language промпт.
  • at(key, lang, **kwargs) — локализованные строки с подстановкой.
  • get_calc_questions, get_credit_menu_buttons, get_main_menu_buttons.
intent.py
rule-based intent
Regex/substring-хелперы распознавания интентов.
  • _is_greeting, _is_thanks, _is_operator_request.
  • _is_calc_trigger(text) — кнопка «Рассчитать» (только <40 символов).
  • _is_recalculate, _looks_like_question, _detect_product_category.
  • _is_currency_question — требует явных USD/EUR/валют (не ловит «курс» как подстроку).
products.py
продукты · БД + форматирование
Загрузка и форматирование банковских продуктов.
  • _get_products_by_category(category) — загрузка из БД, агрегация, rate_matrix/rate_schedule.
  • _find_product_by_name(name, products) — fuzzy-поиск.
  • _format_product_list_text, _format_product_card — HTML-форматтеры.
calc_extractor.py
LLM parse · numbers
LLM-извлечение числовых значений из ответа пользователя.
  • extract_calc_value(...) → {"type":"value"/"question", "value":..., "_usage":...}.
  • extract_prefill_from_history(...) — pre-fill calc_slots из истории.
  • extract_updated_value(...) — detect «поменяй сумму на X».
parsers.py
utils · no-LLM
Утилиты парсинга строк/ответов (чистый Python).
checkpointer.py
persistence · LangGraph
Инициализация async checkpointer для persistence LangGraph state.
  • _create_async_checkpointer(backend, url):
    • auto/postgres → AsyncPostgresSaver из DATABASE_URL.
    • иначе → MemorySaver.
lang_detect.py
детекция языка
Детекция языка пользователя один раз per turn до запуска графа.
  • detect_language(text, fallback) — главный entry.
  • _should_skip_detection, _fast_path_detect (uz-Cyrillic + morphology regex), _normalize_detector_output.
  • _DETECTOR_SYSTEM_PROMPT — DECISION ORDER: «русская грамматика перевешивает общую банковскую лексику».
pii_masker.py
PII · regex masker
Regex-замена PII на токены перед отправкой в OpenAI.
  • mask_pii(text) — последовательно: EMAIL → IBAN → PHONE (+998) → CARD (16) → PINFL (14) → PASSPORT (2+7).
  • Идемпотентно (токены не матчат паттерны). Даты и ИНН не трогаются.
nodes/router.py
роутер · Command(goto)
Минимальная маршрутизация на основе state.
  • node_router(state) возвращает Command(goto=..., update=...).
  • 4 ветки: human_mode / calc_flow (lead или flow=calc) / calc_flow (кнопка «Рассчитать») / faq.
nodes/faq.py
LLM + tools · ≤3 раунда
Главный FAQ-узел. LLM видит 12 инструментов и выбирает действие.
  • Нормализация текста, XML со state в system prompt, trim истории до 12 сообщений.
  • Loop ≤3 раза: llm.ainvoke → tool_calls → ToolNode.ainvoke → results.
  • Fallback: _faq_lookup из app/utils/faq_tools.py на APIError/timeout.
  • _update_dialog_from_tools — обновление dialog по последнему tool_call.
  • _reattach_keyboard — context-appropriate кнопки (по flow).
nodes/calc_flow.py
калькулятор · lead
Двухфазный flow: calc_step (сбор параметров) → lead_step (захват контактов).
  • _handle_calc_step — extract value → clamp → следующий вопрос → результат (PDF/текст).
  • _handle_lead_step — offer → name → phone, _save_lead_async() в БД.
  • Helpers: _lookup_credit_rate, _lookup_deposit_rate, _clamp_term, _clamp_downpayment.
nodes/human_mode.py
оператор · interrupt
Прерывание графа для ввода оператора.
  • node_human_mode_turn(state) — langgraph_interrupt(...), возврат пустого answer.
  • Resume: Agent.resume_human_mode() → Command(resume=operator_reply).
nodes/helpers.py
утилиты · finalize
Вспомогательные функции для финализации turn и persistence.
  • _finalize_turn(state, answer, dialog, keyboard, is_fallback, mask_user_text) — PII-mask, append messages, trim, fallback_streak, show_operator_button.
  • _save_lead_async(data) — создание Lead в БД.

Замечания и гоча

  1. Свежий SystemMessage каждый turn в _ainvoke — без этого lang «замораживался» бы в checkpointer, и смена языка пользователем не применялась бы.
  2. Fast-path в lang_detect экономит LLM-вызов и устраняет главный класс ошибок (uz в русских буквах → ru).
  3. PII внутри node_faq. Двойная защита: детектор + _finalize_turn. Но пользовательский текст не маскируется перед HumanMessage(content=normalized_text or user_text) на faq.py:240 — PII потенциально уходит в основной LLM в рамках turn'а (маскируется только в истории после _finalize_turn). Стоит проверить, критично ли.
  4. Tool-call loop ≤3 раундов — защита от «заклинивания» LLM; выход по timeout с is_fallback=True и увеличением fallback_streak.
  5. Operator button автоматически показывается после 3 подряд fallback'ов (FALLBACK_STREAK_THRESHOLD) или при явном запросе оператора.
  6. Checkpointer auto тихо падает на MemorySaver если Postgres недоступен — в проде стоит явно пинать LANGGRAPH_CHECKPOINT_BACKEND=postgres, иначе reboot = потеря state.
  7. Router _is_calc_trigger — только строки <40 символов; длинные сообщения с «рассчитать» внутри уходят в faq, LLM сам решит через start_calculator().
  8. Telegram router теперь не имеет substring-фолбэков — весь произвольный текст идёт в агент. Кнопки меню (action == CURRENCY_RATES и т.д.) обрабатываются на уровне handler'а.
